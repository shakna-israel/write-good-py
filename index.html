write-good-py

This is a Python Implementation of Brian Ford's write-good, a naive grammar linter for the English language.

This implementation is currently under heavy development and not recommended for use, development or otherwise.

Why?

I love write-good, and I use it often for both prose and documentation.

In automating some documentation building, I wanted to incoroporate some automated checks, using write-good, however, as the documentation engine was Python, I wanted something of the same flavour to make everything simpler.

Usage

If you must use it, it follows a pretty familiar interface for those used to the command-line:

python write_good.py -i YourFile.txt

Options can be seen by running:

python write_good.py
